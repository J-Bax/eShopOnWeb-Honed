The `CatalogItemNameSpecification` at `CatalogItemNameSpecification.cs:10` filters by name:

This generates `SELECT COUNT(*) FROM Catalog WHERE Name = @p0`. The `Name` column has a max length of 50 (`CatalogItemConfiguration.cs:19`) but no dedicated index — the only index is the primary key on `Id`. As items accumulate during the load test (each iteration creates a new item), the table grows and the unindexed scan becomes progressively slower.

## Theory

`COUNT(*)` must scan all matching rows to produce an exact count, whereas `EXISTS` (generated by `AnyAsync`) can short-circuit after finding the first match. With the load test creating hundreds of items, and no index on `Name`, each create request triggers a full table scan. Using `AnyAsync` instead of `CountAsync` allows the query to stop at the first matching row, and the Ardalis.Specification `RepositoryBase` supports `AnyAsync` natively.

1. **Replace `CountAsync` with `AnyAsync`:** Change line 44 from `CountAsync(catalogItemNameSpecification)` to `AnyAsync(catalogItemNameSpecification)`, and update the condition on line 45 to check the boolean result directly. This generates an `EXISTS` query that short-circuits.
